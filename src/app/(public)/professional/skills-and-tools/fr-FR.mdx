## Bienvenue !

Depuis l'âge de 12 ans, je suis en contact avec la technologie, mais je me concentrais davantage sur le côté **design**. J'ai travaillé avec de nombreuses **technologies et outils différents**, ce qui m'a appris _comment les choses fonctionnent et comment les utiliser à mon avantage_.

Aujourd'hui, j'ai trouvé ma passion pour le côté **développement**, à la fois front-end et back-end. Je suis toujours à la recherche de nouvelles technologies et outils à apprendre et à utiliser dans mes projets, alors jetez un œil à la liste ci-dessous pour voir avec quoi je travaille actuellement. :)

---

## Développement Back-end

Je m'épanouis dans le développement back-end, où mon objectif est de créer des solutions **robustes, efficaces et évolutives**. Pour cela, je travaille actuellement avec les technologies suivantes :

### Langages de Programmation

#### Node.js (JavaScript/TypeScript)

En tant que principale technologie backend, Node.js me permet de créer des applications réseau rapides et évolutives. Je l'utilise pour créer des API, des microservices et de la logique côté serveur.

#### Ruby

Actuellement en transition de Node.js à Ruby, j'utilise Ruby pour créer des applications web robustes et évolutives.

### Frameworks & Bibliothèques

#### Express

Un framework d'application web minimal et flexible pour Node.js. Je l'utilise pour créer des API, des applications web et de la logique côté serveur.

#### NestJS

Un framework progressif pour Node.js, conçu pour construire des applications côté serveur efficaces, fiables et évolutives.

#### Ruby on Rails

Un framework d'application web qui inclut tout ce qui est nécessaire pour créer des applications web basées sur des bases de données selon le modèle MVC (Modèle-Vue-Contrôleur).

#### SQLite

J'utilise SQLite pour de petits projets et des tests. C'est une bibliothèque en langage C qui implémente un moteur de base de données SQL petit, rapide, autonome, fiable et complet. Pour cela, j'utilise la bibliothèque suivante :

### ORMs & ODMs

#### Mongoose

Lorsque je travaille avec MongoDB, j'utilise Mongoose. C'est un outil de modélisation d'objets pour MongoDB conçu pour fonctionner dans un environnement asynchrone.

#### Typegoose

Lorsque j'utilise TypeScript avec MongoDB, j'utilise Typegoose. C'est une bibliothèque TypeScript qui ajoute une typage statique aux modèles Mongoose.

#### Sequelize

En tant qu'ORM pour Node.js, Sequelize est un ORM pour Node.js basé sur les promesses pour Postgres, MySQL, MariaDB, SQLite et Microsoft SQL Server.

### Authentification

#### OAuth2

OAuth 2.0 est le protocole standard de l'industrie pour l'autorisation. Je l'utilise pour fournir un accès délégué sécurisé aux ressources du serveur.

#### JWT

Le JSON Web Token (JWT) est une norme ouverte qui définit un moyen compact et autonome de transmettre des informations de manière sécurisée entre des parties en tant qu'objet JSON. Je l'utilise pour l'authentification et l'autorisation.

## Développement Front-end

Ma partie préférée du développement est le front-end. J'adore créer des **interfaces utilisateur belles et interactives** qui offrent une **expérience utilisateur fluide**. Cela étant dit, je travaille actuellement avec les technologies suivantes :

### Balises, Styles & Scripts

#### HTML5

Le langage de balisage standard pour les documents conçus pour être affichés dans un navigateur web.

#### CSS3

Le langage de feuille de style utilisé pour décrire la présentation d'un document écrit en HTML. J'utilise les bibliothèques suivantes avec CSS3 :

#### JavaScript (TypeScript)

Le langage de programmation qui permet des pages web interactives et est une technologie essentielle pour le développement web moderne.

### Bibliothèques & Frameworks

#### React

Ma bibliothèque JavaScript préférée pour construire des interfaces utilisateur. C'est une bibliothèque déclarative, efficace et flexible qui me permet de créer des interfaces utilisateur complexes avec facilité.

#### Next.js

En tant que framework React, Next.js permet des fonctionnalités telles que le rendu côté serveur (SSR) et la génération de sites statiques pour des applications web basées sur React.

#### Vue

Comme React, Vue est un framework JavaScript pour la création d'interfaces utilisateur. Comme React, il est déclaratif, efficace et flexible.

#### Nuxt

Comme Next.js, il permet des fonctionnalités telles que le rendu côté serveur (SSR) et la génération de sites statiques pour des applications web basées sur Vue.

#### Tailwind CSS

Mon approche préférée pour les styles. C'est un framework CSS utilitaire-first pour la création rapide de designs personnalisés. Il me permet de créer des designs beaux et réactifs avec facilité.

#### Sass (SCSS)

Si je n'utilise pas Tailwind CSS, j'utilise Sass pour mes styles. Il me donne plus de puissance et de flexibilité lorsque j'écris des styles.

### Tests

#### Jest

Mon outil de test préféré. Je l'utilise pour tester du code JavaScript, y compris les composants React et Vue. Jest est un framework de test JavaScript agréable axé sur la simplicité.

## Outils

En tant que développeur, j'utilise beaucoup d'outils pour faciliter ma vie. Voici quelques-uns des outils que j'utilise quotidiennement :

### Productivité

#### Obsidian

Pour la prise de notes et la gestion des connaissances, j'utilise Obsidian. C'est une base de connaissances puissante, largement personnalisable. Je l'utilise pour suivre mes projets, idées et notes.

#### Notion

Notion fonctionne comme mon outil de gestion des tâches. C'est un espace de travail tout-en-un, mais je l'utilise principalement pour suivre mes projets et tâches.

#### Visual Studio Code Insiders

En tant qu'éditeur de code principal, Visual Studio Code Insiders est l'endroit où je passe la plupart de mon temps à coder. Il est polyvalent, personnalisable et dispose d'une grande communauté avec de nombreuses extensions et contenus pour m'aider dans mes projets.

### Design

#### Figma

Figma est mon outil de design préféré. C'est un éditeur de graphiques vectoriels basé sur le web et un outil de prototypage qui permet à plusieurs personnes de collaborer en temps réel. Je l'utilise pour créer des wireframes, maquettes et prototypes pour mes projets.

#### Photoshop et Illustrator

Les deux produits Adobe sont mes outils préférés pour l'édition d'images et les graphiques vectoriels. Je les utilise pour créer et éditer des images, des illustrations et des icônes pour mes projets. Je les utilise également pour créer des assets pour mes sites web et applications.

### Systèmes d'exploitation

#### Windows

Mon système d'exploitation principal. Je l'utilise pour des usages généraux, le jeu et parfois même pour coder. C'est un excellent système d'exploitation pour la productivité et le divertissement.

#### Arch Linux

Mon système d'exploitation de prédilection pour coder. Arch Linux est une distribution Linux légère et flexible qui essaie de rester simple. Je l'utilise principalement pour le développement, car c'est ma distribution Linux préférée.

### DevOps

#### Docker

Une plateforme pour développer, expédier et exécuter des applications dans des conteneurs. Je l'utilise pour créer, déployer et exécuter des applications en utilisant des conteneurs. C'est un excellent outil pour automatiser le déploiement des applications.

#### Kubernetes

Kubernetes est un système open-source d'orchestration de conteneurs pour automatiser le déploiement, la mise à l'échelle et la gestion des applications conteneurisées dans un environnement en cluster.
