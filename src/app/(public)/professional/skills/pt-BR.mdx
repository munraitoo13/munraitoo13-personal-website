## Bem-vindo!

Desde os 12 anos, estou em contato com a tecnologia, mas meu foco era mais voltado para o lado **do design**. Trabalhei com várias **tecnologias e ferramentas diferentes**, e isso me ensinou _como as coisas funcionam e como usá-las a meu favor_.

Hoje em dia, encontrei minha paixão no lado **do desenvolvimento**, tanto no front-end quanto no back-end. Estou sempre em busca de novas tecnologias e ferramentas para aprender e usar em meus projetos, então dê uma olhada na lista abaixo para ver com o que estou trabalhando atualmente. :)

---

## Desenvolvimento Back-end

Eu prospero no desenvolvimento back-end, onde meu objetivo é criar **soluções robustas, eficientes e escaláveis**. Para isso, atualmente estou trabalhando com as seguintes tecnologias:

### Linguagens de Programação

#### Node.js (JavaScript/TypeScript)

Como minha principal tecnologia de back-end, o Node.js me permite construir aplicações de rede rápidas e escaláveis. Eu o utilizo para criar APIs, microsserviços e lógica do lado do servidor.

#### Ruby

Atualmente migrando do Node.js para Ruby, estou utilizando Ruby para construir aplicações web robustas e escaláveis.

### Frameworks & Bibliotecas

#### Express

Um framework web minimalista e flexível para Node.js. Eu o uso para construir APIs, aplicações web e lógica do lado do servidor.

#### NestJS

Um framework progressivo para Node.js, projetado para construir aplicações do lado do servidor eficientes, confiáveis e escaláveis.

#### Ruby on Rails

Um framework de aplicação web que inclui tudo o que é necessário para criar aplicações web baseadas em banco de dados de acordo com o padrão Model-View-Controller (MVC).

### Bancos de Dados

#### MongoDB

#### PostgreSQL

#### SQLite

Uso SQLite para pequenos projetos e propósitos de teste. É uma biblioteca escrita em C que implementa um motor de banco de dados SQL pequeno, rápido, autônomo e de alta confiabilidade. Para isso, uso a seguinte biblioteca:

### ORMs & ODMs

#### Mongoose

Ao trabalhar com MongoDB, uso Mongoose. É uma ferramenta de modelagem de objetos MongoDB projetada para funcionar em um ambiente assíncrono.

#### Typegoose

Se estou usando TypeScript com MongoDB, uso Typegoose. É uma biblioteca TypeScript que adiciona tipagem estática aos modelos Mongoose.

#### Sequelize

Como um ORM para Node.js, Sequelize é um ORM baseado em promessas para Postgres, MySQL, MariaDB, SQLite e Microsoft SQL Server.

### Autenticação

#### OAuth2

OAuth 2.0 é o protocolo padrão da indústria para autorização. Eu o uso para fornecer acesso delegado seguro a recursos do servidor.

#### JWT

O JSON Web Token (JWT) é um padrão aberto que define uma maneira compacta e autônoma de transmitir informações de forma segura entre as partes como um objeto JSON. Eu o utilizo para propósitos de autenticação e autorização.

## Desenvolvimento Front-end

Minha parte favorita do desenvolvimento é o front-end. Adoro criar **interfaces de usuário bonitas e interativas** que proporcionam uma **experiência de usuário perfeita**. Dito isso, atualmente estou trabalhando com as seguintes tecnologias:

### Marcação, Estilos & Scripts

#### HTML5

A linguagem padrão de marcação para documentos projetados para serem exibidos em um navegador da web.

#### CSS3

A linguagem de folha de estilos usada para descrever a apresentação de um documento escrito em HTML. Eu uso as seguintes bibliotecas com CSS3:

#### JavaScript (TypeScript)

A linguagem de programação que possibilita páginas web interativas e é uma tecnologia essencial para o desenvolvimento web moderno.

### Bibliotecas & Frameworks

#### React

Minha biblioteca JavaScript preferida para construção de interfaces de usuário. É uma biblioteca declarativa, eficiente e flexível que me permite criar interfaces complexas com facilidade.

#### Next.js

Como um framework React, Next.js habilita funcionalidades como SSR e a geração de sites estáticos para aplicações web baseadas em React.

#### Vue

Assim como React, Vue é um framework JavaScript para construção de interfaces de usuário. Assim como React, é declarativo, eficiente e flexível.

#### Nuxt

Assim como Next.js, habilita funcionalidades como SSR e a geração de sites estáticos para aplicações web baseadas em Vue.

#### Tailwind CSS

Minha abordagem favorita para estilos. É um framework CSS utilitário para construção rápida de designs personalizados. Ele me permite criar designs bonitos e responsivos com facilidade.

#### Sass (SCSS)

Se não estiver usando Tailwind CSS, uso Sass para meus estilos. Ele me fornece mais poder e flexibilidade ao escrever estilos.

### Testes

#### Jest

Meu testador preferido. Eu o uso para testar código JavaScript, incluindo componentes React e Vue. Jest é um framework de testes JavaScript focado na simplicidade.

## Ferramentas

Como desenvolvedor, uso muitas ferramentas para facilitar minha vida. Aqui estão algumas das ferramentas que uso diariamente:

### Produtividade

#### Obsidian

Para anotações e gerenciamento de conhecimento, uso o Obsidian. É uma base de conhecimento poderosa e altamente personalizável. Eu o utilizo para acompanhar meus projetos, ideias e anotações.

#### Notion

O Notion funciona como minha ferramenta de gerenciamento de tarefas. É um espaço de trabalho tudo-em-um, mas eu o utilizo principalmente para acompanhar meus projetos e tarefas.

#### Visual Studio Code Insiders

Como meu editor de código principal, o Visual Studio Code Insiders é onde passo a maior parte do meu tempo programando. É versátil, personalizável e possui uma grande comunidade com muitas extensões e conteúdo para me ajudar em meus projetos.

### Design

#### Figma

O Figma é minha ferramenta de design preferida. É um editor de gráficos vetoriais baseado na web e uma ferramenta de prototipagem que permite a várias pessoas colaborarem em tempo real. Eu o utilizo para criar wireframes, mockups e protótipos para meus projetos.

#### Photoshop e Illustrator

Ambos os produtos da Adobe são minhas ferramentas preferidas para edição de imagens e gráficos vetoriais. Eu os utilizo para criar e editar imagens, ilustrações e ícones para meus projetos. Também os utilizo para criar recursos para meus sites e aplicações.

### Sistemas Operacionais

#### Windows

Meu sistema operacional principal. Eu o uso para propósitos gerais, jogos e, às vezes, até para programar. É um ótimo SO para produtividade e entretenimento.

#### Arch Linux

Meu sistema operacional preferido para programação. O Arch Linux é uma distribuição Linux leve e flexível que tenta manter as coisas básicas. Eu o utilizo principalmente para desenvolvimento, pois é minha distribuição Linux preferida.

### DevOps

#### Docker

Uma plataforma para desenvolver, enviar e rodar aplicações em containers. Eu a utilizo para criar, implantar e rodar aplicações usando containers. É uma ótima ferramenta para automatizar a implantação de aplicações.

#### Kubernetes

O Kubernetes é um sistema de orquestração de containers open-source para automação de implantação, escalonamento e gerenciamento de aplicações em containers. Eu o utilizo para gerenciar aplicações containerizadas em um ambiente clusterizado.
